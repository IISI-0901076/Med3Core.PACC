手動測試指南 — Docker Desktop 本機環境

前置條件

  • Windows 10/11 已安裝 Docker Desktop（已啟動）
  • .NET SDK 10.0 已安裝（僅需 build 時）
  • 專案路徑：C:\NHI\claude_nhi3\三代\PACC\Backend\

Step 1：啟動服務

1 開啟 PowerShell，切換到專案目錄並啟動 Docker Compose 
cd C:\NHI\claude_nhi3\三代\PACC\Backend
docker compose up -d --build
💡 說明：首次啟動需要 build image，約需 1-3 分鐘。後續啟動會使用快取。

服務架構：

服務Port說明
postgres5432PostgreSQL 16 資料庫
api5100PACC WebAPI (.NET 10)
smoke-test—自動化測試（啟動時自動跑）

Step 2：確認服務就緒

2 等待約 10 秒後，執行健康檢查 
curl.exe http://localhost:5100/PACC/health
✅ 預期回傳：Healthy
⚠️ 若回傳錯誤：等待 15 秒再試，或執行 docker compose logs api 檢查日誌。

Step 3：測試 10 個 API 端點

📌 Base URL：http://localhost:5100/PACC/pacc1000/PACC1001N01/
📌 成功判斷：回傳 JSON 中 "rtnCode": "0" 即表示成功。
POST① CreateCase — 建立案件
curl.exe -X POST http://localhost:5100/PACC/pacc1000/PACC1001N01/CreateCase ^
  -H "Content-Type: application/json" ^
  -d "{\"apId\":\"IPLA0001\",\"hospId\":\"0101010010\",\"funcType\":\"1\",\"caseType\":\"A\"}"

📝 回傳的 caseSeqNo 值請記下，後續步驟需要使用。

POST② QueryStatus — 查詢案件狀態
curl.exe -X POST http://localhost:5100/PACC/pacc1000/PACC1001N01/QueryStatus ^
  -H "Content-Type: application/json" ^
  -d "{\"caseSeqNo\":\"1\"}"
POST③ UpdateCaseStatus — 更新案件狀態
curl.exe -X POST http://localhost:5100/PACC/pacc1000/PACC1001N01/UpdateCaseStatus ^
  -H "Content-Type: application/json" ^
  -d "{\"caseSeqNo\":\"1\",\"pgmId\":\"IPLA0001\",\"caseStatus\":\"1\"}"
POST④ CreateAttFile — 新增附件檔案
curl.exe -X POST http://localhost:5100/PACC/pacc1000/PACC1001N01/CreateAttFile ^
  -H "Content-Type: application/json" ^
  -d "{\"caseSeqNo\":\"1\",\"itemNo\":1,\"fileName\":\"test.pdf\",\"fileSize\":1024}"
POST⑤ UpdateFileLocation — 更新檔案位置（以 S3 Object Key 為 Key）
curl.exe -X POST http://localhost:5100/PACC/pacc1000/PACC1001N01/UpdateFileLocation ^
  -H "Content-Type: application/json" ^
  -d "{\"objectKey\":\"pacc/2026/test.pdf\",\"fileStatus\":\"1\",\"s3Bucket\":\"nhi-pacc\"}"
POST⑥ QueryCaseFiles — 查詢案件檔案清單
curl.exe -X POST http://localhost:5100/PACC/pacc1000/PACC1001N01/QueryCaseFiles ^
  -H "Content-Type: application/json" ^
  -d "{\"caseSeqNo\":\"1\"}"
POST⑦ CreateRecvLog — 建立接收紀錄
curl.exe -X POST http://localhost:5100/PACC/pacc1000/PACC1001N01/CreateRecvLog ^
  -H "Content-Type: application/json" ^
  -d "{\"caseSeqNo\":\"1\",\"apId\":\"IPLB0001\",\"hospId\":\"0101010010\"}"

📝 回傳的 recvSeqNo 值請記下。

POST⑧ UpdateRecvLog — 更新接收紀錄
curl.exe -X POST http://localhost:5100/PACC/pacc1000/PACC1001N01/UpdateRecvLog ^
  -H "Content-Type: application/json" ^
  -d "{\"recvSeqNo\":\"1\",\"procResult\":\"0\",\"procMsg\":\"OK\",\"fileSize\":2048,\"procTime\":\"20260522164200\",\"totalCount\":10}"
POST⑨ LogError — 記錄錯誤
curl.exe -X POST http://localhost:5100/PACC/pacc1000/PACC1001N01/LogError ^
  -H "Content-Type: application/json" ^
  -d "{\"caseSeqNo\":\"1\",\"pgmId\":\"IPLA0001\",\"errorCode\":\"E001\",\"errorMsg\":\"Test error\"}"
POST⑩ UpdateCaseFiles — 更新案件檔案傳送狀態
curl.exe -X POST http://localhost:5100/PACC/pacc1000/PACC1001N01/UpdateCaseFiles ^
  -H "Content-Type: application/json" ^
  -d "{\"caseSeqNo\":\"1\",\"fileSendStatus\":\"1\"}"

常見問題排除

Q: 啟動後 health check 回傳連線失敗？

A: API 需等待 PostgreSQL 就緒（約 5-10 秒）。若超過 30 秒仍失敗：

docker compose logs api
docker compose logs postgres

Q: 回傳 "rtnCode": "500"？

A: 查看 API 日誌取得詳細錯誤：

docker compose logs api --tail 50

Q: Port 5100 已被佔用？

A: 修改 docker-compose.yml 中 api 服務的 ports 設定：

# 改為其他 port，例如 5200
ports:
  - "5200:8080"

Q: 想重新初始化資料庫？

docker compose down -v
docker compose up -d
